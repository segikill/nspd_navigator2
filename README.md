# NSPD Navigator for QGIS Kolba

Модифицированный webscript для QGIS Kolba: подложки НСПД, базовые карты, выгрузка объектов по охвату, выгрузка тайлов ортофото в переносимый VRT и служебные `metadata.json`.

## Установка через Kolba

В Kolba откройте получение Web Script и укажите raw-ссылку:

```text
https://raw.githubusercontent.com/segikill/nspd_navigator2/main/nspd_navigator.py
```

После сохранения скрипт появится в списке инструментов Kolba. Для проверки обновлений Kolba использует поле `original_url` из верхнего docstring скрипта.

## Основные функции

- Подключение тематических слоев НСПД из `layers-theme-tree?themeId=1`.
- Подключение базовых карт НСПД из `/api/geoportal/v1/baselayers`.
- Папка `Базовые карты` с подложками: ЕЭКО ортофото, ЕЭКО, ДЗЗ РФ, цифровая объектовая схема, ортофотопланы 2000 и 10000.
- Добавление WMS/WMTS-подложек в проект QGIS двойным кликом по слою.
- Сохранение WMS/XYZ-соединений в Browser/Connections QGIS.
- Выбор охвата на карте для выгрузки векторных/объектных данных НСПД.
- Пакетная загрузка объектов по тайлам с повторами при временных ошибках.
- Выгрузка растровых WMTS-подложек по выбранному охвату и выбранному zoom `z`.
- Сохранение выгрузки растра в отдельную папку: `VRT + tiles/ + metadata.json`.
- Переносимый VRT: тайлы подключаются относительным путем `tiles/...`, поэтому папку можно переносить на другой ПК целиком.
- Отображение текущего масштаба карты и расчетного `z≈` в панели выбранного слоя.
- Статус и прогресс выгрузки: количество тайлов, проценты, повторы, ошибки, zoom.
- Защита от случайной загрузки растров на слишком мелком масштабе, чтобы не перегружать QGIS.
- Обход проблем SSL/подключения НСПД в новых версиях QGIS, включая QGIS 3.44.

## Как пользоваться

1. Запустите скрипт из Kolba.
2. Выберите слой в дереве.
3. Для обычных тематических слоев нажмите кнопку выгрузки и выделите охват на карте.
4. Для базовых карт/WMTS выберите охват, задайте zoom `z` и папку сохранения.
5. После выгрузки откройте созданный `.vrt` в QGIS. Внутри папки рядом будут `tiles/` и `metadata.json`.

## Метаданные выгрузки

`metadata.json` содержит служебную информацию:

- id и имя слоя;
- zoom `z`;
- время начала и окончания выгрузки;
- диапазон тайлов;
- количество скачанных тайлов, ошибок и повторов;
- URL источника слоя;
- дату обновления слоя НСПД, если она есть в API;
- кандидаты дат из URL источника, если они распознаны.

Важно: `system_updated`, `Last-Modified` и кандидаты дат из URL не считаются подтвержденной датой съемки.

## Обновление версии

При каждом изменении скрипта нужно:

1. Обновить `nspd_navigator.py`.
2. Повысить поле `version:` в первом docstring.
3. Проверить, что raw-ссылка открывает чистый Python-код, а не HTML-страницу GitHub.
4. В Kolba выбрать скрипт и нажать Update, если версия отличается.
